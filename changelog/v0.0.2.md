# Gitwig 0.0.2

English | [한국어](./v0.0.2.ko.md)

Gitwig `0.0.2` adds a terminal panel and improves branch cleanup, tag creation, large change review, and update guidance.

## New

- Select multiple branches in the sidebar with Shift or Command, then delete eligible local or remote branches together.
- Create annotated tags from history and push them to a selected remote.
- Open an embedded terminal panel powered by SwiftTerm, with the bundled D2Coding font for clearer command-line output.

## Changed

- Resize unstaged, staged, and commit sections when reviewing large change sets.
- Use multi-file actions from the working changes list to stage, unstage, stash, discard, copy paths or names, and reveal files in Finder.
- See tags more reliably in the sidebar, including tags that are not near the currently loaded history.
- Get clearer update guidance when a newer Gitwig version is available or when an update is required.

## Fixes

- Changed-file selection now stays visible while navigating long commit or stash file lists.
- Merge summary rows no longer expand on the first selection click, reducing accidental expansion while browsing history.
- Moving between the sidebar, history, file browser, and working changes now clears old highlights more predictably.
- Empty file detail views are less distracting when no file content, history, or blame information is available.
